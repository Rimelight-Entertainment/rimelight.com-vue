# Oxlint Custom Rule Wishlist

Oxlint currently does not have support for custom linting rules. This is a record of custom rules to be added in the future.

---

## Incorrect usage of Ref within non-component context
**Severity:** Error

**Rationale:** Nuxt SSR can lead to cross-request state pollution. Use useState() or a Pinia store instead.

### Incorrect

```typescript
// composable.ts

const cartItems = ref<CartItem[]>([])
export const useCart = () => {
return { cartItems }
}
```

### Correct

```typescript
// composable.ts

const cartItems = useState<CartItem[]>('cart_items', () => [])
export const useCart = () => {
return { cartItems }
}
```

## Component Prop Interface and Destructuring
**Severity:** Warning

**Rationale:** Component props must be exported as an interface and declared with destructuring.

### Incorrect

```typescript
// Component.vue

const props = withDefaults(defineProps<{
    name: string
}>(), {
    name: "Component"
})
```

### Correct

```typescript
// Component.vue

export interface ComponentProps {
    name: string
}

const { name = "Component" } = defineProps<ComponentProps>()
```
